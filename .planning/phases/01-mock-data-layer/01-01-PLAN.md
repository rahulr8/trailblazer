  <action>
Create `lib/mock/` directory and `lib/mock/types.ts` with all UI-specific TypeScript interfaces.

Define these 9 interfaces using ONLY primitive types (string, number, boolean, null, arrays, string literal unions). NEVER import from `firebase/`, `@/lib/db/`, or `@/lib/health/`.

**Interfaces to define (use research examples as reference):**

1. **MockUser** - id, displayName, photoURL, membershipTier ("free" | "platinum"), email
2. **MockStats** - totalMinutes, totalKm, totalSteps, currentStreak, longestStreak, totalActivities, totalBadges, totalChallenges (all numbers)
3. **MockReward** - id, vendor, title, description, featured (boolean), rewardType ("qr" | "barcode" | "code"), rewardValue, color, imageUrl (all strings except featured)
4. **MockLeaderboardEntry** - rank, userId, displayName, avatarUrl, totalTime (number, in minutes), isCurrentUser (boolean)
5. **MockAchievement** - id, name, description, iconName, earned (boolean), earnedDate (string | null for ISO 8601 or null if not earned)
6. **MockHeroCard** - id, type ("motivation" | "counter"), motivationText (optional string), minutesActive (optional number)
7. **MockPulseItem** - id, userId, displayName, avatarUrl, activityType, message, timestamp (ISO 8601 string)
8. **MockQuest** - id, title, description, progress (number 0-100), targetValue, currentValue (numbers), unit (string like "km", "minutes", "activities")
9. **MockWalletCard** - id, name, type ("membership" | "pass" | "partner"), barcodeType ("qr" | "barcode"), barcodeValue, color, logoUrl

**Conventions (from CONVENTIONS.md):**
- Use `export interface` for all shapes (not `type`)
- PascalCase interface names with `Mock` prefix
- Use string literal unions for discriminated fields
- Dates as ISO 8601 strings (never Firebase Timestamp)
- Non-null types for happy-path UI state (except earnedDate which is genuinely nullable for unearned achievements)
